# Visual Rhythm

Use this before building a full HTML deck, especially for long source material or report-to-deck conversions.

## Principle

A polished deck needs one visual system but many page jobs. Repetition should feel intentional, not like the agent pasted every section into the same card/list layout.

## Rhythm Plan

Before writing HTML, make a compact rhythm table for the whole deck:

- slide or section
- slide job: hook, map, chapter, mechanism, evidence, process, comparison, checklist, detail, synthesis
- visual form: full-bleed image, ledger, matrix, timeline, flow, map, quote, table, dashboard, modal/detail layer
- implementation choice: AI image, SVG/process diagram, chart/map, CSS texture, typography-only, no media
- slot status: not needed, reserved, generated, inserted, verified
- media role: generated image, real image, CSS texture, chart, diagram, typography-only
- motion recipe: reveal, path draw, count-up, tab, toggle, zoom/pan, none
- density: speaker-led, mixed, reading/detail

If three consecutive slides share the same visual form and density, redesign at least one of them.

## Media Decision

Pick the clearest explanation tool after the outline and style direction are known:

- AI image: abstract atmosphere, reusable chapter backplate, spatial metaphor, or low-risk subject texture.
- SVG/process diagram: exact logic, named steps, scoring rules, loops, dependencies, and flow relationships.
- Chart/map/table: measurable evidence, thresholds, rankings, geospatial position, and comparisons.
- CSS texture or typography-only: pacing, emphasis, title pages, and dense content where extra imagery would distract.

Start HTML authoring as soon as the rhythm table is stable. For planned images that are not generated yet, reserve a fixed-size slot and mark its status as `reserved`; continue building nearby SVG diagrams, charts, interactions, and detail layers.

## Long Source Material

When the source must not be shortened:

- Put the live-talk version on the main slide: claim, 3-5 evidence points, implication.
- Preserve full source text in a detail layer, appendix slide, speaker note, or continuation slide.
- Make the detail affordance obvious, but do not make it visually compete with the main argument.
- Avoid shrinking text below comfortable reading size to satisfy "no deletion".

## Anti-Monotony Patterns

Rotate among these within one visual system:

- chapter opener with one big number and one sentence
- route map or learning map
- process diagram with path-draw motion
- two-column comparison
- ledger or checklist
- risk matrix
- timeline or phased sequence
- evidence/detail slide with expandable source
- synthesis loop or final operating model

Use at least four distinct page roles for any deck longer than 12 slides.

## Generated Image Use

Generated images should create rhythm, not fill gaps:

- cover hero: establishes mood and subject
- section texture: distinguishes chapters
- mechanism visual: clarifies system relationships
- transition visual: connects two sections

For formal report decks, prefer abstract backplates, maps, terrain/radar textures, and diagram-ready backgrounds over fake documentary scenes.

## Visual Rhythm QA

Before delivery, scan screenshots as thumbnails:

- The viewer can tell which slides are section openers, content pages, detail pages, and synthesis pages.
- Important pages do not all have the same left-title/right-card layout.
- Generated images are visible where intended but do not hide text.
- Motion recipes differ by meaning, not randomly by page.
